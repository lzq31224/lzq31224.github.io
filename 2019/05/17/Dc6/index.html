DC6渗透记录

  1. 主机发现

    $ netdiscover

    得到192.168.105.143这个ip

  2. 扫描端口

  1. 访问80端口，可以看到 http://wordy/，却无图像显示
    我们添加一下hosts文件

  2. 用7kb扫描目录，得知是wordpress的站点，用wpscan枚举用户

    得到五个用户，将用户写入user.txt

  3. 接下来切一份密码字典出来

  4. 接下来，开始用wpscan跑密码
    wpscan –url http://wordy/ -U user.txt -P pass.txt

  5. 接下来用密码登陆wordpres
    这个wordpress是有一个cve的，利用方法

  6. kali里有一个自带的模块也可以查找exp

  7. 接下来，开始利用此漏洞（已知此漏洞可以命令执行）

    先点击Activity monitor然后在点击tool 填入数值后，点击lookup，然后burp抓包

    可以看出，此处可以进行命令执行，接下来，构造反弹shell


    监听成功，接下来，我们需要获取一个交互式shell

  8. 开始提权
    发现suid无法提权，于是开始随意翻找，在/home/mark/stuff/things-to-do.txt中找到一个账号和密码

    用ssh登陆进去


    查看这个用户能执行的文件

    找到一个backups.sh，可以执行，而且还有读写权限的文件，而且，改文件的所属是jens用户，所以，我们可以切换到jens执行此文件,我们可以用此文件反弹回jens用户的权限，

    命令成功执行，接下来，我们用nc监听反弹回的会话（先监听，后执行）


    查看jens可以执行的程序


    发现一个nmap，使用root执行的，而且不需要root密码，我们可以使用nmap提权
    先写一个nmap插件nse

    接下来用nmap加载这个插件
    $ sudo nmap –script=/home/jens/fuck_root.nse

    可以看到，已经是root权限了，
